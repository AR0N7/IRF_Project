# IRF_Project

Egyelőre csak vázlatos:

Az ötlet: Bejelentkezés után (unit test), MNB webszolgáltatást felhasználva (legalább egy új függvény), valamiféle valutaváltó app (xml), ahol lehet venni/eladni/kereskedni, stb, megfelelő időközökkel (timer) tranzakciók között. A végén a napi eredményt csv-be exportálni is lehet. Közben legyen több unit test is.

Külön formon a current ráták, korábbi ráták elemzése, diagramm!!
Külön valuta vásárló formon kiválasztani az összes currency közül amit venni szeretne, a mennyiséget (ami után automatikusan kiírja az árat mindkét valutában) és megvétel gombbal listához adni (Unit test bele, csak olyan valuták, amik megvannak)

+10 pontért megpróbálni: fejlesztési minta (Pattern.txt file is kell)

Ebben az esetben a projektnek tartalmaznia kell legalább egy olyan absztrakt osztályt vagy interfészt, amit legalább két másik osztályban felhasználásra kerül. Ha valaki szeretne ezzel megpróbálkozni, akkor mindenképp egyeztessen előtte az egyik gyakorlatvezetővel, mutassa be ötletét és az alkalmazni kívánt mintát. Körülbelül egy oldalas specifikáció szükséges. Egyeztetés hiányában a fejlesztési mintáért plusz pont nem adható.

Legalább 3 saját osztály vagy származtatott vezérlő (1 már van)
Olyan összetett legyen, mint a Sudoku
15-20 órát foglalkozni vele (8-9 nap, napi 2 óra kb) - Eddig foglalkoztam vele: 4,5 óra
5 perces videó a kódról!
